## Accomplishments

1. **PTY session persistence** — Terminal sessions now survive browser reloads. Server-side registry with 30-min TTL, scrollback replay on reconnect, and `reconnecting` status chip with amber pulse animation.
2. **Canvas panel extracted to persistent column** — Live Canvas moved from a sidebar tab to an always-visible resizable right column with localStorage persistence and a toolbar toggle button.
3. **Bottom panel shell** — Collapsible panel below the main session area, separated by a horizontal drag resize handle. Height and open/closed state persisted to localStorage.
4. **Raw terminal sessions** — Multi-tab PTY shells in the bottom panel. Each tab is an independent process; supports add/close, auto-seed on project open, kill-on-unmount for cleanup.
5. **Canvas MCP + multi-tab canvas** — Claude CLI gets first-class canvas access via 5 MCP tools (`canvas_open`, `canvas_update`, `canvas_lock`, `canvas_unlock`, `canvas_close`). Canvas panel upgraded to a multi-tab interface with force-close guard for locked tabs.
6. **MCP connections UI** — Toolbar button + modal for managing MCP server registrations. One-click `slopmop-canvas` auto-registration via `claude mcp add`. Per-server status dots, descriptions, and remove buttons.

### Phase 13: Raw Terminal Sessions

**Goal:** Populate the bottom panel with plain PTY terminal sessions — no Claude agent, just a raw shell. Supports multiple tabs with add/close, each tab is an independent PTY process. Reuses existing usePty infrastructure.

**Requirements:** RAWTERM-01, RAWTERM-02, RAWTERM-03, RAWTERM-04, RAWTERM-05, RAWTERM-06
**Plans:** 3/3 complete

| Plan | Focus | Outcome |
|------|-------|---------|
| 13-01 | Wave 0 RED stubs + useRawSessionManager hook | Hook GREEN, MAX_RAW_SESSIONS=4 |
| 13-02 | RawTerminalPane component + App.tsx wiring (tab bar, auto-seed, body) | Component GREEN |
| 13-03 | CSS for tab chips/add button + human visual verify | Human verified |

Key decisions:
- No localStorage persistence for raw sessions — ephemeral by design
- `display:none` panel isolation keeps xterm/WebSocket state alive across panel hide/show
- `killOnUnmount` option in usePty distinguishes ephemeral raw terminals (kill) from persistent agent sessions (detach)
