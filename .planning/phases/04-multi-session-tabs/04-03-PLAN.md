**Architecture: one SessionPane per session (React hooks rules compliant)**

Each session gets its own `SessionPane` component that calls `useSession` once. This is the only architecture that satisfies React hooks rules without conditional calls. The App calls `useSessionManager` once and maps sessions to `<SessionPane>` elements.

**Step 1: Create client/components/SessionPane.tsx**

SessionPane is an internal component. It calls useSession once, owns the terminal mount, editor tabs, and attachments for a single session. It receives all session-level state via props.

```typescript
interface SessionPaneProps {
  sessionId: string;
  cwd: string;
  agentConfig: AgentConfig;
  isActive: boolean;
  onStatus: (status: SessionStatus) => void;
  onExit: (code: number) => void;
  onFirstInput: (sessionId: string, text: string) => void;
  // SessionPane renders the TerminalComponent, EditorTabBar, Composer, etc.
  // i.e., everything currently in App.tsx's "active session" render path
}
```

Inside SessionPane:
- Call `useSession({ cwd, sessionId, agentConfig, onStatus, onExit, ... })`
- Render `<TerminalComponent>`, `<EditorTabBar>`, `<Composer>`, `<FilePreview>`, etc. (move from App.tsx)
- Wrap the entire output in: `<div style={{ display: isActive ? 'flex' : 'none', flex: 1, flexDirection: 'column', minHeight: 0 }}>`
- For first-prompt name extraction: track a `hasNamedRef = useRef(false)` inside SessionPane. On the first non-empty `sendInput` call, call `onFirstInput(sessionId, text)` before forwarding to session.sendInput. Set hasNamedRef.current = true after first call so subsequent inputs do not trigger onFirstInput.

The `extractSessionName` utility:
```typescript
function extractSessionName(raw: string): string {
  return raw.replace(/\r$/, '').replace(/\x1b\[[0-9;]*[mGKH]/g, '').trim().slice(0, 40) || 'Session';
}
```

4. Replace single terminal block with mapped SessionPanes:
```tsx
<div className="terminal-area" onClick={() => composerRef.current?.focus()}>
  <SessionTabBar
    sessions={sessionManager.sessions}
    activeId={sessionManager.activeId}
    onSetActive={sessionManager.setActive}
    onClose={sessionManager.close}
    onSpawn={() => { if (cwd) sessionManager.spawn(cwd); }}
  />
  {sessionManager.sessions.map(s => (
    <SessionPane
      key={s.id}
      sessionId={s.id}
      cwd={s.cwd}
      agentConfig={agentConfig}
      isActive={s.id === sessionManager.activeId}
      onStatus={(status) => sessionManager.updateStatus(s.id, status)}
      onExit={(code) => sessionManager.updateStatus(s.id, code === 0 ? 'done' : 'error')}
      onFirstInput={(id, text) => sessionManager.updateName(id, extractSessionName(text))}
    />
  ))}
</div>
```

PITFALL GUARD: Use `key={s.id}` (stable UUID) on SessionPane — never derive the key from index or anything that changes.

PITFALL GUARD: Only call sendResize inside the active SessionPane. SessionPane should only call `session.sendResize` when `isActive` is true, or pass `isActive` into useSession so it can gate resize events.

PITFALL GUARD: When cwd changes (new folder selected), do NOT auto-spawn on top of existing sessions. Clear the sessions array first (or treat a new cwd as a fresh manager state), then call spawn once.
