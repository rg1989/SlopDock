Replace the single `useSession` call in App.tsx with `useSessionManager`-driven multi-session rendering.

**Key changes:**

1. Import useSessionManager and SessionTabBar:
```typescript
import { useSessionManager } from './hooks/useSessionManager';
import { SessionTabBar } from './components/SessionTabBar';
```

2. Replace the single `useSession` call. The problem with replacing it directly is that useSession owns editor tabs, attachments, and PTY per session. The cleanest approach for v1: keep useSession for the ACTIVE session only, but use useSessionManager to track the sessions array, names, and statuses.

   Implementation strategy:
   - useSessionManager owns: sessions[], activeId, spawn, close, setActive, updateName, updateStatus, history
   - The existing `useSession` call remains for the active session only (as today), but receives `onStatus` and `onExit` callbacks that feed into useSessionManager.updateStatus
   - Each session gets its own xterm.js Terminal instance, stored in a Map<sessionId, Terminal> ref
   - On cwd set, call sessionManager.spawn(cwd) to create the first session

3. Per-session terminal map:
```typescript
const terminalMapRef = useRef<Map<string, Terminal>>(new Map());
const handleTerminalReady = useCallback((sessionId: string, t: Terminal) => {
  terminalMapRef.current.set(sessionId, t);
  if (sessionId === sessionManager.activeId) setTerminal(t);
}, [sessionManager.activeId]);
```

4. Replace single `<TerminalComponent>` with mapped sessions:
```typescript
{sessionManager.sessions.map(s => (
  <div
    key={s.id}
    style={{ display: s.id === sessionManager.activeId ? 'flex' : 'none', flex: 1, minHeight: 0 }}
  >
    <TerminalComponent
      onReady={(t) => handleTerminalReady(s.id, t)}
      sendResize={s.id === sessionManager.activeId ? session.sendResize : undefined}
    />
  </div>
))}
```

5. Add SessionTabBar above the terminal-area div:
```tsx
<div className="main-area">
  <div className="terminal-area" onClick={() => composerRef.current?.focus()}>
    <SessionTabBar
      sessions={sessionManager.sessions}
      activeId={sessionManager.activeId}
      onSetActive={(id) => {
        sessionManager.setActive(id);
        const t = terminalMapRef.current.get(id);
        if (t) setTerminal(t);
      }}
      onClose={(id) => sessionManager.close(id)}
      onSpawn={() => { if (cwd) sessionManager.spawn(cwd); }}
    />
    {/* mapped session terminals */}
    ...
  </div>
  ...
```

6. Wire onStatus into useSession/usePty. Extend useSession's UsePtyOptions passthrough to accept sessionId and onStatus. In usePty call inside useSession, pass:
```typescript
sessionId: activeSessionId,
onStatus: (status) => sessionManager.updateStatus(activeSessionId, status),
onExit: (code) => sessionManager.updateStatus(activeSessionId, code === 0 ? 'done' : 'error'),
```

7. First-prompt name extraction: wrap the `session.sendInput` passed to Composer. After the first non-empty message per session, call `sessionManager.updateName(activeId, extractSessionName(text))`. Track first-sent state with a ref keyed by sessionId.

extractSessionName utility (add as a local function in App.tsx or import from useSessionManager):
```typescript
function extractSessionName(raw: string): string {
  return raw.replace(/\r$/, '').replace(/\x1b\[[0-9;]*[mGKH]/g, '').trim().slice(0, 40) || 'Session';
}
```

PITFALL GUARD: Do NOT use session.id as the React key for TerminalComponent — the key must be stable and must not change. Use the session id from sessionManager.sessions (UUID, stable).

PITFALL GUARD: Only call sendResize on the active session's terminal. Pass `undefined` for sendResize when s.id !== activeId to avoid inactive PTY resize calls that missize inactive terminals.

PITFALL GUARD: When cwd changes (new folder selected), do NOT auto-spawn a new session on top of existing ones for the new cwd. The spawn should only happen once per handleConnect call. Clear sessions array (or let useSessionManager handle cwd change) and spawn fresh.
